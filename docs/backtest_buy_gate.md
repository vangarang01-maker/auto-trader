# 매수 게이트 백테스트 — RSI 임계 조정 (35 → 45)

## 배경

자동매매가 가동 이후 **한 번도 실제 매수 체결되지 않은** 문제를 추적한 결과,
인프라·키·코드 버그가 아니라 **매수 게이트와 종목 선정 전략의 모순**이 원인으로 확인됨.

- **스크리너**(V1/V2/V3): health_score·모멘텀 기반 **강세·우량주** 선정 → RSI 평소 40~70+
- **매수 게이트**(`src/portfolio/manager.py`): `RSI < 35`(과매도) **AND** `거래량 ≥ 20일평균 × 1.5`
- 강세주가 과매도(RSI<35)까지 떨어지는 일이 거의 없어 → **매수 신호가 구조적으로 발생하지 않음**

GitHub Actions `trade.yml` 실행 로그에서도 매 회차 전 종목 `[대기]`(RSI 40~82, 35 미만 0개)로 확인됨.

## 백테스트 방법

`backtest_gate.py` — 현재 picks 14종목, 최근 6개월(2025-12-01 ~ 2026-06-05) 일봉.

- RSI-14: Wilder smoothing (`calc_rsi`와 동일 방식)
- 거래량비: `당일 거래량 / 직전 20일 평균`
- 수익 시뮬: 신호일 종가 진입 → 익절 +15% / 손절 -7% / RSI ≥ 75 청산
- 데이터 소스: `FinanceDataReader`

## 해석

1. **기존 게이트는 6개월 × 14종목 통틀어 신호 1건** — 매매 미발생을 수치로 확정.
2. **RSI 임계 완화** → 거래 빈도 증가. `< 45`가 적정(6개월 11매매).
3. **거래량 AND 조건이 신호 품질의 핵심** — 제거 시 승률 91% → 57%로 폭락(잡신호 대량 유입).

→ **`RSI < 45` AND `거래량 1.5x` 채택.** 적정 빈도 + 승률 91% + 평균 +17%. 거래량 게이트는 품질필터로 유지.

## 한계 (과적합 주의)

- 표본 구간이 **강세장**(KOSPI 200MA 대비 +60~82%) → 승률 과대평가 가능. 약세장 미검증.
  (단 `run_trade.py`의 `bear_market` 신규매수 차단으로 일부 방어.)
- 14종목 = **현재** picks → 생존편향. 과거 실제 picks 구성과 다름.
- 슬리피지·체결 지연·`MAX_HOLD=5` 한도 미반영.
- **상대 비교** 목적으로는 유효(기존 게이트 거래 불가 입증 + 거래량 AND의 품질 기여 확인).
